## 생성자 패턴

<p align><img width="400" src="./images/생성자 패턴.png"></p>

- 생성자는 객체가 새로 만들어진 뒤 초기화하는 데 사용되는 메서드
- ES2015 버전 이후 생성자를 가진 클래스를 만들 수 있게 되었고, 생성자를 통해 클래스의 인스턴스 객체 생성

### 객체 생성 및 값 할당

```tsx
// 객체 생성
// 리터럴 표기
const obj_a = {}
// Object.create 메서드
const obj_b = Object.create(Object.prototype)
// new 키워드
const obj_c = new Object()

// 할당
obj_a["name"] = "KIM"
obj_b.name = "PARK"
Object.defineProperty( obj_c , "name", {
	value: "LEE"
	writable: true,
	enumberable: true,
	configurable: true
})

// 객체 상속
const guest = Object.create(obj_a);
guest.defineProperty( obj_c , "age", {
	value: 20
	writable: true,
	enumberable: true,
	configurable: true
})

```

- 객체는 리터럴, `Object.create` 메서드, new 키워드를 통해 생성 가능
- dot(.) 문법, 대괄호 문법, `Object.defineProperty` 를 통해 속성을 할당 가능
- `Object.create(객체)`를 통해 객체를 상속 가능

### 생성자의 기본 특징

```tsx
class Person {
  constructor(lastName, firstName, age, assets) {
    this.lastName = lastName;
    this.firstName = firstName;
    this.age = age;
    this.assets = assets;
  }

  getFullName() {
    return this.lastName + this.firstName;
  }
}

let HP = new Person('Harry', 'Porter', 20, 1000);
let KM = new Person('Kim', 'MinJae', 24, 100000);

console.log(HP); // PerSon { lastName: 'Harry', firstName: 'Porter', age: 20, assets: 1000}
console.log(HP.getFullName()); // HarryPorter
```

- 클래스는 새 객체를 초기화하는 `contructor()`라는 이름의 메서드를 가지고 있어야 한다.
- new 키워드를 통해 생성자를 호출할 수 있고, `this` 키웓는 새로 생성된 해당 객체를 가리킨다.
- 이러한 생성자 패턴에서 여러 문제 발생
  - 상속이 어려움
  - `Person` 생성자로 객체 생성 시 매번 `getFullName()` 과 함수를 새로 정의

### 프로토타입을 가진 생성자

```tsx
class Person {
  constructor(lastName, firstName, age, assets) {
    this.lastName = lastName;
    this.firstName = firstName;
    this.age = age;
    this.assets = assets;
  }
}

Person.prototype.getFullName = function () {
  return this.lastName + this.firstName;
};

let HP = new Person('Harry', 'Porter', 20, 1000);
let KM = new Person('Kim', 'MinJae', 24, 100000);
console.log(HP.getFullName()); // HarryPorter
console.log(KM.getFullName()); // KimMinJae
```

- 생성자의 프로토타입 객체에 속한 속성을 새로운 객체(인스턴스)에서 활용함으로써, 동일한 프로토타입 객체를 사용하는 여러 개의 `Person` 객체 생성 가능
