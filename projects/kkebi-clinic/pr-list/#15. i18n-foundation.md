## #15 : ko/en 다국어 처리 구현

작업 링크 📎 : https://github.com/SungHyun627/KKEBI-clinic/commit/b293eb74

### 💡 다음 이슈를 해결했어요.

- Next.js 다국어(i18n) 흐름을 전반적으로 안정화했습니다.
  - `ko/en` 전환 버튼 동작 보완
  - 404 / 에러 페이지(`not-found`, `error`, `global-error`) 문구를 번역 키 기반으로 통일
  - 메인 레이아웃 하드코딩 문구(기본 사용자명, 로그아웃 실패 메시지, 이동 aria 등) i18n 적용
- 내담자 관리/상세 드로어 및 관련 플로우를 보완했습니다.
  - 알림 드로어 `즉시 확인`, 주간통계 `확인하기` 클릭 시 `/clients` 상세 드로어 오픈 흐름 개선
  - `clientId` query 반응성 개선 (`useSearchParams` 기반)
  - 동일 내담자 재진입 시에도 열리도록 `openAt` query 키 도입
- 인증/알림 관련 메시지 혼입(한/영 섞임) 방지를 위해 UI 노출 문구를 번역 키 중심으로 정리했습니다.
- 빌드 차단 이슈를 해결했습니다.
  - Next.js Route export 형식 오류(유효하지 않은 named export) 수정
  - auth API route들을 `export async function POST(...)` 형태로 통일
  - `next build --webpack` 기준 빌드 통과 확인

### 💡 이슈를 처리하면서 추가된 코드가 있어요.

- 번역 키 추가
  - `common`: `defaultUserName`, `logoutFailed`, `goToDashboardAria`, `underConstruction`, `backToDashboard`, `sessionStartTitle`, `sessionStartDescription`, `inputAssistiveButton`, `inputClearContent`
  - `notFound`, `error` 섹션 키 정리/추가
- clients query drawer 상태 제어 로직 개선
  - dismiss 기준에 `clientId + openAt` 키 사용
- 위험 알림 이동 URL 개선
  - `RiskNotificationItem`, `RiskAlert`에서 `openAt` query 추가
