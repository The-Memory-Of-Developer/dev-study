## #2 : 프론트엔드 개발 환경 설정

작업 링크 📎 : https://github.com/SungHyun627/KKEBI-clinic/commit/206f46d

### 💡 다음 이슈를 해결했어요.

- 프론트엔드 개발 환경 초기 세팅을 진행했습니다.
- Husky를 적용해 커밋 시점에 코드 품질 체크가 동작하도록 설정했습니다.
- Git hook 기반으로 팀 공통 개발 규칙을 강제할 수 있는 기반을 마련했습니다.
- `package.json`과 lockfile을 정리해 로컬/CI 환경에서 동일한 의존성 상태를 유지할 수 있도록 구성했습니다.

### 💡 이슈를 처리하면서 추가된 코드가 있어요.

- `.husky/pre-commit` 추가
  - 커밋 전에 사전 검증이 실행되도록 pre-commit 훅을 설정했습니다.
- `package.json` 수정
  - Husky 실행을 위한 `prepare` 스크립트 구성을 반영했습니다.
- `pnpm-lock.yaml` 갱신
  - 의존성/스크립트 변경 사항을 lockfile에 반영했습니다.

### 💡 필요한 후속작업이 있어요.

- pre-commit 단계에서 실행할 검사 항목(lint, format, test) 범위를 팀 기준으로 확정할 필요가 있습니다.
- CI와 로컬 훅 동작이 동일한지 점검하고, 필요 시 워크플로우와 정합성을 맞출 예정입니다.
