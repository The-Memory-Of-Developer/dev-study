### 💡 다음 이슈를 해결했어요.

- 세션 알림 발송 Drawer를 구현하고 주요 진입점에 연결했습니다.
  - 대시보드 오늘의 일정(`TodayScheduleAction`)

  - 상담 세션 카드(`ScheduledSessionItemCard`)
- 세션 알림 발송 기능을 API와 연결했습니다.
  - 발송 조건(알림 유형 선택 시 활성화)
  - payload 전송(다음 상담 일정, 알림 유형, 메시지 내용)

  - 성공 토스트에 수신자 이름 포함(한글: `이름 + 공백 + 님` 형식)
- 내담자 상세 정보 수정/종결 기능을 구현했습니다.
  - 상세 정보 수정 진입/저장 흐름
  - 종결 다이얼로그 및 종결 API 연결
  - 종결 시 활성 목록에서 즉시 제거 + 토스트 처리
- 종결 상담자 목록 페이지(`clients/closed`)를 구현했습니다.
  - 리스트/빈 상태/복구 버튼
  - 복구 API 연결
  - 종결 사유 칩 분리
  - 다국어 처리(종결 사유, 주 호소 문제, 내담자명 매핑)

- 현재 알림 채널 중 `push`, `sms`는 UI에서 disabled 처리되어 있어 실제 채널 API 연동이 필요합니다.
