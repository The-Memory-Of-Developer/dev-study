## #4 : 로그인, 2FA, 비밀번호 재설정, 상담사 등록 문의 Flow 구현

작업 링크 📎 : https://github.com/SungHyun627/KKEBI-clinic/commit/c96bc31

### 💡 다음 이슈를 해결했어요.

로그인, 2FA, 비밀번호 재설정, 상담사 등록 문의까지 인증 관련 핵심 Flow를 구현했습니다.

**로그인 Flow 구현**

- 로그인 Form UI를 구현하고 이메일/비밀번호 유효성 검증을 적용
- 로그인 성공/실패 분기 처리와 실패 Dialog를 구현
- 로그인 정보 저장(로그인 유지) 로직을 localStorage 기반으로 적용
- auth/main 라우팅 및 공통 layout 구조를 정리해 인증 화면 진입 흐름을 안정화

**2FA 인증 Flow 구현**

- 2FA 인증 Form UI를 구현
- 인증 코드 입력/수정 로직과 “새 코드 요청” 기능을 구현
- 인증 실패 시 재확인 모달, 인증 성공 시 대시보드 이동 흐름을 적용
- 인증하기 버튼 submit 처리 등 폼 동작을 보정

**비밀번호 재설정 Flow 구현**

- 비밀번호 재설정 페이지를 구현
- 비밀번호 변경 요청 후 이메일 발송 완료 모달 노출 로직을 구현
- 입력한 이메일을 발송 완료 모달 문구에 반영하도록 개선
- 비밀번호 재설정 관련 UI/로직을 분리해 유지보수성 향상

**상담사 등록 문의 Flow 구현**

- 상담사 등록 문의 모달 UI를 구현
- 문의 접수 완료 모달을 구현해 접수 완료 경험을 명확히 제공

**공통 UI/구조 개선**

- 공통 Input/Button/Label/Textarea 컴포넌트를 정비
- 로그인 폴더 구조 리팩토링 및 widgets/features 분리를 적용
- 기능별 demo/server API 분리 및 demo 기본 연결 구조를 적용

### 💡 이슈를 처리하면서 추가된 코드가 있어요.

- 인증 도메인 중심으로 페이지/컴포넌트/로직이 추가
- 로그인/2FA/비밀번호 재설정/문의 접수 모달 관련 UI 컴포넌트가 추가
- 인증 단계별 상태 처리(성공, 실패, 재요청, 완료)에 필요한 분기 로직이 추가
- 공통 컴포넌트(Input, Label, Textarea 등)와 auth 레이아웃/라우팅 구성 정리
- demo/server API 분리 구조가 반영되어 이후 실제 API 연동 전환 기반을 마련

### 💡 필요한 후속작업이 있어요.

- demo 기본 연결 상태를 실제 백엔드 API 연동으로 전환할 필요가 있습니다.
- 인증/비밀번호 재설정/문의 접수 Flow에 대한 E2E 테스트 보강이 필요합니다.
- 에러 메시지/문구의 i18n 적용 범위를 확대할 필요가 있습니다.
