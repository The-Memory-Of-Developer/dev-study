## #44 : 상담 종료 complete API 연동 및 summary 상세 이동/제출완료 분기 반영

작업 링크 📎 : https://github.com/SungHyun627/KKEBI-clinic/commit/f4e21900

### 💡 다음 이슈를 해결했어요.

- 상담 세션 종료 시 요약 페이지로 이동하기 전에 세션 종료 전용 API를 호출하도록 흐름을 변경했어요.
  - `상담 종료 버튼 -> 녹음 업로드 -> POST /sessions/{sessionId}/complete -> summary 페이지 이동`
- summary 페이지에서 `isSubmitted`를 제출 완료 플래그로 해석하도록 UI/동작을 정리했어요.
  - 제출 완료 상태에서는 수정/재제출이 불가능하도록 처리
  - 뒤로가기 시 "작성 중지" 확인 다이얼로그가 뜨지 않도록 분기
- 완료된 세션 카드의 `View Details(상세보기)` 버튼으로 summary 페이지 이동이 가능하도록 연결했어요.

### 💡 이슈를 처리하면서 추가된 코드가 있어요.

- BFF 라우트 추가
  - `src/app/api/v1/sessions/[sessionId]/complete/route.ts`
- 세션 종료 API 클라이언트 모듈 추가
  - `src/features/session/api/completeSessionById.ts`
- 연동/분기 반영 파일
  - `src/features/session/ui/page/SessionPageContent.tsx`
  - `src/features/summary/hooks/useSessionSummary.ts`
  - `src/features/summary/ui/SessionSummaryContent.tsx`

  - `src/features/summary/ui/components/SummaryTopBar.tsx`
  - `src/features/sessions/session-list/ui/CompletedSessionItemCard.tsx`

### 💡 필요한 후속작업이 있어요.

- `/complete` 호출 실패 시 사용자 재시도 UX(버튼/다이얼로그) 보강 여부 논의가 필요해요.
- 백엔드 반영 직후 `GET /summary` 응답 지연 가능성이 있으면 summary 초기 조회 재시도(폴링) 정책 검토가 필요해요.
