## #48 : 테스트 커버리지 확대 및 CI 테스트 파이프라인 분리

작업 링크 📎 : https://github.com/SungHyun627/KKEBI-clinic/commit/cea49b62

### 💡 다음 이슈를 해결했어요.

- 테스트 커버리지를 확대해서 리팩토링 구간의 회귀를 빠르게 감지할 수 있도록 구성했습니다.

**단위/통합 테스트 보강**

- `http-client`의 `401 -> refresh -> retry`, `403` 분기 테스트를 추가했습니다.
- 세션 시작 API 성공/실패 분기 테스트를 추가했습니다.
- 세션 요약 제출 DTO 매핑/실패 분기 테스트를 추가했습니다.
- i18n `ko/en` 메시지 키 동등성 회귀 테스트를 추가했습니다.
- `useSessionInsightsStream` 재연결/정리 훅 테스트를 추가했습니다.
- 자동기록 패널 통합 테스트(SSE 반영, 스페이스 단축키)를 추가했습니다.
- SSE 이벤트 파싱 유틸 테스트를 추가했습니다.

**세션 실시간 기능 테스트 구조 개선**

- `SessionAutoRecordPanel` 내부 파싱/정규화 로직을 유틸로 분리하고 테스트 가능한 구조로 변경했습니다.
- 감정/인지왜곡/PHQ 파싱, payload unwrapping, transcript upsert parsing 같은 순수 로직을 분리했습니다.

**E2E 스모크/크리티컬 시나리오 강화**

- 스모크 시나리오(`auth.login`, `session.realtime`, `session.sse-reconnect`, `summary.submit`)를 추가/보강했습니다.
- 크리티컬 시나리오(`app.user-flow`, `en.routes`) 핵심 UI 단언을 강화했습니다.
- Playwright 5xx 응답 감지 헬퍼를 추가해서 “화면은 보이지만 서버 에러가 난” 케이스도 실패로 검증하도록 했습니다.

**E2E 네이밍/실행 규칙 통일**

- 파일 네이밍을 `domain.scenario.tier.spec.ts` 규칙으로 통일했습니다.
- `*.critical-path.spec.ts`를 `*.critical.spec.ts`로 통일했습니다.
- `app-critical-user-flow`, `en-critical-routes`는 규칙에 맞는 신규 파일로 재구성했습니다.
- `test:e2e:smoke`, `test:e2e:critical` 실행 대상을 패턴 기반으로 정리했습니다.

**CI 파이프라인 개선**

- 기존 테스트 작업을 `unit`, `e2e-smoke`, `e2e-critical`로 분리했습니다.
- Playwright 리포트를 smoke/critical 각각 아티팩트로 업로드하도록 구성했습니다.
- Playwright 브라우저 설치/실행 경로 불일치 이슈를 수정했습니다.
- smoke/critical 스크립트 정규식 오류(`Invalid regular expression`)를 수정했습니다.

**최신 화면/라우팅 반영**

- `clients/closed -> clients/terminated` 리다이렉트를 테스트 단언에 반영했습니다.
- strict mode 충돌이 발생하던 타이틀 단언(`Weekly stats`, `주간 통계`)을 보정했습니다.
- CI에서 `API base URL` 미설정 시 발생하던 500 회귀를 공통 인증 mock 확장으로 방지했습니다.
